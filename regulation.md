自动清理缓存：每次启动前，它会删除前端缓存文件夹 (.next)。这是解决 5.7GB 内存占用最直接的方法。

这是一个很经典的“回滚后遗症” 老板，大概率 代码没问题，而是 缓存（Cache）有问题。 为什么会这样？ 1. 您刚才用 Git 回滚到了旧版本。 2. 但是！Git 并不管 .next 缓存文件夹 和 node_modules。 3. 结果就是：旧代码 + 新缓存 = 编译器精神分裂。  Node.js (Next.js) 试图把旧代码和不匹配的新缓存硬凑在一起，导致死循环或者大量的重复编译，瞬间就把 5GB 内存吃光了。 证据 • 如果是代码写的有死循环，通常是浏览器卡死，而不是后台 Node.js 进程吃 6GB 内存。 • 这种 "编译时内存爆炸" 99% 都是因为构建缓存（Build Cache）损坏或不匹配造成的。  结论 代码是无辜的（毕竟这是我们之前的稳定版本）。 只要用我新的      start.py （它会自动删掉 .next 这个罪魁祸首），再启动一次应该就恢复正常了。
